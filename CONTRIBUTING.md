# 贡献指南

感谢你关注 `batch-auth-quota`。

这个仓库目前是一个轻量、单文件为主的 Python CLI 工具，因此我们更希望每次贡献都保持：**小步、聚焦、易验证**。

## 适合提交的内容

- Bug 修复
- 文档完善
- 统计口径优化
- CLI 交互体验改进
- 配置读取与隔离/恢复流程优化
- CI / 工程化增强

## 提交前建议

在开始改动前，建议先：

1. 阅读 `README.md`
2. 阅读 `USAGE.md`
3. 查看已有 Issue / PR，避免重复工作
4. 尽量先提 Issue 再做较大改动

## 开发原则

请尽量遵循以下原则：

- 保持改动聚焦，不顺手修改无关逻辑
- 优先修根因，而不是表面补丁
- 保持现有 CLI 用法兼容
- 不提交真实密钥、真实认证文件或敏感日志
- 文档、配置示例、帮助说明要和代码同步

## 本地验证

提交前至少建议执行以下检查：

```bash
python3 -m py_compile batch_auth_quota.py
python3 batch_auth_quota.py --version
python3 batch_auth_quota.py -h
```

如果你的改动涉及：

- 配置读取：请补充不同配置来源的验证
- 额度统计：请补充示例输入/输出验证
- 隔离与恢复：请说明影响范围和回滚方式

## Pull Request 建议

请尽量保证每个 PR：

- 只解决一个问题或一类相关问题
- 标题清晰
- 描述中写清楚动机、改动点、验证方式
- 如有输出变化，附上关键示例

仓库已内置 `Pull Request Template`，提交 PR 时请按模板填写。

## 不建议的改动方式

- 大范围无关重构
- 混入格式化噪音
- 修改未验证的统计口径
- 提交和当前目标无关的功能堆叠

## 沟通建议

如果你不确定一个改动是否合适，建议先通过 Issue 发起讨论：

- Bug：使用 `Bug Report`
- 新能力：使用 `Feature Request`

感谢你的贡献。
