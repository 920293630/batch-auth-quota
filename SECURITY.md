# 安全说明

## 安全边界

`batch-auth-quota` 是一个本地运行的 CLI 工具，主要处理：

- 本地认证文件
- 管理密钥
- CLI Proxy API 管理接口
- 额度统计输出文件

由于涉及认证文件与管理密钥，使用时请特别注意敏感信息保护。

## 不要提交的内容

请不要在 Issue、PR、日志或截图中公开以下信息：

- `CPA_MANAGEMENT_KEY`
- `MANAGEMENT_PASSWORD`
- 真实认证文件内容
- `access_token` / `refresh_token`
- 可直接访问你服务实例的内部地址

如需展示问题，请先做脱敏处理。

## 安全使用建议

- 优先通过 `.env` 管理密钥，不要硬编码到脚本或公开配置中
- 不要把真实认证文件放进仓库
- 不要把结果目录中的敏感输出直接公开分享
- 在多人机器或共享环境上运行时，注意文件权限
- 对外发布日志前，先检查是否包含邮箱、账号 ID、token 或内部地址

## 漏洞反馈

如果你发现的是**安全问题**，不建议直接公开提交 Issue。

建议方式：

1. 先准备最小复现说明
2. 对敏感信息做脱敏
3. 通过仓库维护者可控的私下方式反馈
4. 在问题修复前，避免公开可被直接利用的细节

当前仓库尚未配置专用安全邮箱，因此建议：

- 先在 GitHub 个人主页或仓库资料中查找联系入口
- 如没有私下联系方式，再提交经过脱敏的最小公开问题描述

## 支持版本

当前优先支持以下代码线：

- `main` 分支
- 最新发布版本（当前为 `v0.1.0`）

如果你使用的是更早的历史版本，建议先升级后再复现问题。

## 支持范围

当前更关注以下安全相关问题：

- 密钥泄露风险
- 敏感文件误输出
- 交互流程导致的误操作
- 隔离/恢复流程中的文件安全问题
- 输出结果中包含过多敏感字段

## 修复原则

安全相关改动通常遵循：

- 优先降低泄露风险
- 默认采用更保守、更安全的行为
- 保持问题可复现、可验证、可回滚
